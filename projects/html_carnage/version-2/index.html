What is HTML and HTML5? - Overview of HTML

HTML stands for "Hypertext Markup Language", in which "hypertext" means a string of texts contains a hyperlink and "markup language" refers to a text-encoding system consisting of a set of symbols inserted in a text document to control its structure, formatting, or the relationship between its parts (From Wikipedia). Another example of markup language is XML (eXtensible Markup Language).

Simply put, HTML is a markup language for hypertext. It is a language for creating webpage, defining its structure and giving meaning to different blocks of text for a browser to render. Everything we see in the internet, more or less, contains HTML.

By "marking up" various parts with tags, HTML describes the structure of a webpage and tells User Agents (UAs) - most typical example is a browser, how to render/display the content. Tags define HTML elements and each of them may associate with certain attributes, which some specific whereas some global.

The latest version of HTML is HTML5. It was release in 2014 (the older version was HTML4.1, which was in practice since 1999). Compared with the previous versions, HTML5 introduced many interesting features and is considered much easier to use.

It is important to bear in mind that HTML is the backbone of the web and it has not stopped evolving since then. Along these years, there is ongoing adjustment to HTML in order to match the need following the rapid development of the internet.

Reasons for clearer HTML

Before heading to the reasons for writing HTML in a clear and semantic manner, it is crucial to know SEO.

SEO stands for Search Engine Optimization. The importance of SEO, in particular in Internet Marketing, can be summarized as below,

"SEO is performed because a website will receive more visitors from a search engine when websites rank higher on the search engine results page (SERP). These visitors can then potentially be converted into customers." - Wikipedia

...to be fair, even if it is not for boosting the number of potential customers, it is reasonable that one still wants their webpage to be found and be seen by others - that has been the spirit of internet from very beginning, to interconnect with others.

In past, the algorithm which search engines employed heavily relied on the information provided by the webmasters/content providers. The latter could improve the ranking of their webpages by, for example, adding repeating or even inaccurate search terms / keywords in the metadata. Needless to say more, the search result may appear many irrelevant webpages due to the aforesaid manipulation from the side of webmasters.

To respond such behaviour, search engines started to developing more complex ranking algorithms which calculate the score of webpage in aspect of semantic signals holistically.

That is one of the main reasons and is quite a strong enough reason alone for us to sharpen own HTML skills. HTML may be plain and not very complicated to learn (seemingly). Nonetheless, it is all about semantics which is crucial for SEO and therefore deserves to receive our attention. It is the foundation always being here (as of now, at least).

How to write clearer HTML? (Good practices for HTML)

  1. Avoid abusing <div> and <span>.

    There are tons of alternative with built-in features provided by HTML5 (we will cover that more in other parts on this page).
  2. Be discreet when giving name to the class/ID for elements, be specific (avoid using numbering as class name, ie picture1-6 but picture-left/picture-center).
  3. Have a more compound class attribute with a string of generic classes which could be reuseable - like the way Bootstrap is.
  4. Use lowercase for elements, attributes and also values, for better readability and consistency.
  5. Use labels for labelable elements for better machine readability.
  6. Always think of the possibilities of incompatibility and prepare alternative ways as fall-back.
  7. Use <h1> only once per page.
