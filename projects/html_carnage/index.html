Reasons for cultivating your HTML skills

Such question is to serve as a reminder for myself, so that I would not rush in learning while not having my founcation established solidily. To answer this question, we need to know what is considered good in HTML and among all, writing semantic HTML is very important.

There are several reasons as below:
  1. It makes your code easier to understand so to be easier to update.
  2. It makes the source code more readable for others.
  3. Browsers can interpret your code better and therefore boosts the SEO (Search Engine Optimization), resulted in higher accessibility.

Tricks/Good Practices when writing HTML

  1. Avoid abusing <div> and <span>. There are tons of alternative with built-in features provided by HTML5 (we will cover that more in other parts on this page).
  2. Be discreet when giving name to the class/ID for elements, be specific. (this is difficult for me.. i am never good in naming).
  3. Use lowercase for elements, attributes and also values, for better readability and consistency.
  4. Use labels for bars/options in the form.
  5. Add alt (Alternative Text Attribute) on images/links as fallback.
  6. Use <h1> only once per page.

Semantic Tags

When creating a brief layout, the following tags can be used:

<header> - information related to the title, heading of the related content (with heading elements wrapped), logo/icon, and authorship information.

<footer> - author information (if not in the header), copyright information, contact information, carriers, link back to top or sort.

<article>

<menu> <nav> and <aside>

<section> <details> <summary>

Then, there are some other sematic tags for various types of text/media(image/audio/video) elements: <em> - emphasized <del> - deleted <ins> - inserted <mark> - highlighted <strong> - important <s> - strikethrough <cite> - title of work/publications <dfn> - definition <sup> - Subscript <sub> - Superscript <abbr> - abbreviation <code> - code <kbd> - keyboard <samp> - sample

<p>I was trying to boot my computer, but I got this hilarious message:</p> <p><samp>Keyboard not found Press F1 to continue</samp></p>

Less preferable but yet depreciated and still valid in HTML are: <i> - originally italic, but is suggested to use to denote "technical terms, transiliterations, thoughts, or names of vessels in western texts" without emphasis. <b> - bold <small> - smaller font size

<q> - short quotation, normally comes with quotation marks.

<blockquote> - a block of quotation, like a paragraph or even longer.

<address> - any kind of contact information, including physical address, email, social media account, etc. The text will be in italic and a line break will be before and after the tag.

<figure> <figcaption>

<picture>, <audio> and <video> are new in HTML5, they both require opening tag and closing tag.

Possible attributes are: autoplay, controls, loop, muted, preload and src (which is not essential as it is better to wrap several formats in <source> and <track> and alternative text as fallback).

<track> must be in WebVTT format (.vtt).

The following is an example:

<video width="320" height="240" controls> <source src="forrest_gump.mp4" type="video/mp4"> <source src="forrest_gump.ogg" type="video/ogg"> <track src="fgsubtitles_en.vtt" kind="subtitles" srclang="en" label="English"> <track src="fgsubtitles_jp.vtt" kind="subtitles" srclang="jp" label="Japanese"> Sorry, your browser doesn't support embedded videos. </video>

Note: <source> under <picture> uses srcset instead of src.

Other Useful Tags

<time>

When included to HTML, this can help a user agent offer to add an event to their calendar.

According to MDN, if the element does not have a datetime attribute, it must not have any element descendants, and the datetime value is the element's child text content.

One of the example is: <p>The concert took place on <time datetime="2001-05-15T19:00">May 15</time>.</p>

<map>

For using a single image but mapping to different links. can be useful. <img src="./asakusa_done.svg" alt="Asakusa Shrine - Kaminarimon (Thunder Gate)" usemap="#kaminarimon" width="400" height="379"> <map name="workmap"> <area shape="rect" coords="34,44,270,350" alt="Computer" href="computer.htm"> <area shape="circle" coords="337,300,44" alt="Cup of coffee" href="coffee.htm"> </map>

First add attribute "usemap" and assign an unique ID to the image. then, using <map name={usemap}> to wrap all <area>. Followed with the above, inside <area> set the attributes, the basic ones will be: shape, coords, href and alt.
